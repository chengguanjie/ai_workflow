# 食品配方设计原则（Demo 知识库）

本文件用于 AI Workflow 的 RAG 全流程联调演示（数据为模拟）。

## 1. 配方结构（从目标到配方）

- 目标（Taste / Nutrition / Cost / Process / Shelf-life）优先级要明确
- 约束（过敏原、宗教、素食、控糖、低盐、无麸质）必须在配方层面可追踪
- “主料—辅料—调味—工艺”四层结构便于版本迭代与可解释性

## 2. 常见营养目标的配方策略

### 高蛋白（增肌/健身）

- 主蛋白优先：鸡胸、鱼、虾、蛋清、瘦牛；素食可用豆腐、豆干、毛豆、鹰嘴豆
- 同时配纤维（蔬菜、全谷）提升饱腹感
- 控油：通过酸味（醋/柠檬）、香辛料（蒜、黑胡椒）增强风味而非加油

### 低糖（控糖/减脂）

- 优先降低“游离糖/添加糖”：蜂蜜、糖浆、果酱等
- 需要甜味可选赤藓糖醇等代糖（口感需调和酸味与香味）
- 用“酸 + 香 + 鲜”替代甜味带来的风味厚度

### 低脂

- 控制总油脂，同时避免口感寡淡：优先蒸/煮/焯/烤
- 用“少量油 + 高香味密度”策略：蒜末、葱、香草、烘烤香

## 3. 分量换算（简化规则）

- 家常菜：每人主蛋白 80–120g（高蛋白目标可提升至 120–160g）
- 蔬菜建议：每人 250–400g
- 酱汁盐度：先少后多，避免一次性加盐

## 4. 可解释性（用于 RAG 引用）

配方输出建议包含：

- 为什么这样选主料（目标/约束）
- 为什么这样设定油盐糖
- 引用来源：在回答中标记“来源文档名”
